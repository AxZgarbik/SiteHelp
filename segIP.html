Segmentación


Segmentación por dirección IP


La funcionalidad de orientación de la lista de direcciones IP ha sido reemplazada por la orientación del segmento IP. Encuentre más información sobre la segmentación por segmento aquí:

Subidas de segmentos

Antes de cargar archivos de segmento, primero debe crear segmentos para generar una ID de segmento.

Formato de archivo de segmento
El archivo de segmento debe cumplir con todos los requisitos siguientes:
  1. archivo .txt (no archivo csv)
  2. utf-8 (sin BOM - marcas de orden de bytes) o codificado en ASCII
  3. Una ID de usuario o dirección IP por fila
  4. Pipe delimitado con los ID de segmento en el formato | donde:
    1. <uuid> es el ID de usuario
    2. <segment_ID1> es la clave de segmento que se actualizará en el formato <buzz_key> - <id>
  5. Un tipo de ID de usuario por archivo: Adsmovil Cookie-ID (ADSMOVIL), IDFA, ID de publicidad de Android, dirección IP o ID de cliente (requiere configuración adicional).

Opciones adicionales

  • Puede agregar tantos ID de segmento para los que califique el ID de usuario, siempre que cada uno esté delimitado por tubería después del ID de usuario, por ejemplo, <uuid> | <segment_ID1> | <segment_ID2> | <segment_ID3>
  • Puede agregar un valor de segmento de usuario que se puede utilizar como valor en Bid Modifier . Los valores deben especificarse como un flotante (número), no se aceptarán cadenas, por ejemplo, <uuid> | <segment_ID1>:<value> | <segment_ID2>:<value> | <segment_ID3>

Carga de archivos de segmentos a través de la interfaz de usuario
  1. Sube un archivo de ID de usuario para segmentar asignaciones yendo a Nuevo> Subir segmento.
  2. Declare en la pantalla de carga de segmento si el tipo de ID de usuario es Adsmovil Cookie-ID ("ADSMOVIL"), una de las opciones IDFA / Android, Customer Cookie-ID (requiere configuración adicional) o dirección IP.
  3. Declare el tipo de clave de segmento: elija "PREDETERMINADO" si está utilizando el tipo de identificación de segmento de Adsmovil, a menos que esté utilizando nuestra funcionalidad de identificación alternativa.
  4. Seleccione el continente en el que los usuarios deben estar disponibles para orientar sus anuncios (América, EMEA, APAC). Si los usuarios deberían estar disponibles para segmentar entre regiones, se debería ejecutar una carga separada para cada continente.

Si su archivo de carga tiene más de 10 MB, deberá dividir el archivo para que tenga varios archivos de menos de 10 MB o deberá utilizar nuestro método de carga de archivos de segmentos grandes a través de las cargas de S3 a continuación.

Carga de archivos de segmentos a través de API

La documentación de la API para la carga de archivos de segmento se puede encontrar aquí. Tenga en cuenta que el límite de tamaño de archivo de 10 MB se aplica tanto a las cargas de API como a las cargas de IU.

Carga de archivo de segmento a través de S3

Para cargar archivos de más de 10 MB, los clientes pueden cargar archivos directamente en el depósito AWS S3 de ADSMOVIL. Tenga en cuenta que deberá proporcionar a su administrador de cuentas su ARN de AWS para que ADSMOVIL pueda configurar los permisos de S3 para permitirle colocar los archivos grandes en nuestra nube.

Sobrescribir o eliminar combinaciones de segmentos de usuario

Para eliminar usuarios de su segmento, puede sobrescribir los ID de usuario a través de la API.
