Segmentación


Segmentación por apps / sitio


La segmentación por inventario se utiliza habitualmente para optimizar el rendimiento de sus campañas. En general, la mejor práctica es EXCLUIR la segmentación del inventario que no está funcionando bien para usted. Los Deal ID son una excepción a esta regla general, ya que los Deal ID de acuerdos que suelen incluir el Target.

Dado que la mayoría de segmentación de targeting en el inventario son exchanges con IDs específicos, le pedimos que anteponga la abreviatura del exchange al ID para garantizar la exclusividad. Consulte una lista de las abreviaturas de los intercambios en la página Fuentes de inventario .

Ejemplo: segmentar un ID de sitio de Rubicon
  • La abreviatura de intercambio de Rubicon es "rp"
  • Entonces, para orientar un ID de sitio de "12345" en Rubicon, lo haría especificando "rp / 12345"
Status Adsmovil Significado Recomendacion
App Bundle List

App Bundle list consiste en todos los intercambios. Para iOS, suelen ser el ID numérico de la aplicación en la tienda de aplicaciones, por ejemplo, 1207472156. Para Android, suelen ser cadenas que representan el nombre de la aplicación, como com.game.android.

Para crear una lista:

  1. Haga clic en el botón + Nuevo y seleccione "Lista" en el menú desplegable
  2. Seleccione "App bundle list" en el menú desplegable Tipo de lista
  3. Signe un nombre a su lista y haga clic en "Guardar y continuar"
  4. Agregue sus app Bundle en el campo "Agregar nuevos elementos de lista" (uno por fila) O cargue un .txt de sus ID (también uno por fila) O cargue un archivo .csv de ID (también uno por fila)
 Utilizar como segmentación INCLUDE o EXCLUDE
App ID List

Un ID de aplicación es un ID único específico de intercambio, con el prefijo del código de intercambio. Por ejemplo, en MoPub: mp / 12345

Para crear una lista:

  1. Haga clic en el botón + Nuevo y seleccione "Lista" en el menú desplegable
  2. Seleccione "Lista de ID de aplicación" en el menú desplegable Tipo de lista
  3. Asigne un nombre a su lista y haga clic en "Guardar y continuar"
  4. Agregue sus ID en el campo "Agregar nuevos elementos de la lista" (uno por fila) O cargue un .txt de sus ID (también uno por fila) O cargue un archivo .csv de ID (también uno por fila)
 Utilizar como segmentación INCLUDE o EXCLUDE
App Name El nombre de la aplicación es una descripción de cadena de la aplicación; a menudo está en blanco o no lo proporciona el intercambio. Utilizar como segmentación INCLUDE o EXCLUDE
Deal ID

Deal ID específico de Exchange. Las pujas por deal ID se tratan con mayor prioridad que las pujas de subasta abierta y, por lo general, tienen términos de acuerdos adicionales asociados, incluido un precio mínimo más alto.

Se requiere que anteponga la abreviatura del intercambio, por ejemplo, "rp / 12345"

. DEBE tener permiso explícito para usar un ID de oferta para el editor propietario de la oferta, el intercambio que ofrece la oferta o Adsmovil (si es un PMP utilizable). Si ha descubierto un Deal ID en su panel de Metamarkets , confirme con soporte Adsmovil acerca de su elegibilidad para usar dicho Deal ID

 Utilizar como segmentación INCLUDE o EXCLUDE

Placement ID

Intercambie de ID específico por el placement. Un placement ID se asigna a un ID de etiqueta único en el sitio del editor. Esta es la segmentación más granular del inventario y debe usarse si los ID de dominio o sitio no brindan un control suficientemente detallado.

Se requiere que anteponga la abreviatura del intercambio, por ejemplo, "rp / 12345".

 Utilizar como segmentación INCLUDE o EXCLUDE
